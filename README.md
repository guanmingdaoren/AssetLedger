# 设备资产台账

一个使用 PySide6 构建的本地桌面资产管理程序。所有设备当前信息、分类字典、系统设置和字段级变更历史均保存在同一个 Excel 工作簿中。

## 核心能力

- 每台设备拥有独立的 `AST-年份-序号` 资产 ID。
- 新增、编辑、搜索、排序以及按类别、状态、位置、品牌、来源筛选。
- 责任部门和责任人支持从已有台账选择或直接输入，责任人可用于筛选未分配及具体人员设备。
- 修改状态、位置、价格等字段时自动记录修改前后值。
- 变化记录支持按单个变化字段查看；保存后立即刷新并展示最新历史。
- 每次写入前自动备份，保存时使用临时文件原子替换。
- 设置页维护分类、位置、状态、来源和当前操作者。
- Excel 文件可直接打开查看，正式修改建议通过程序完成。

## 运行

需要 Python 3.11 或更高版本。

```powershell
python -m pip install -r requirements.txt
.\run_asset_ledger.bat
```

也可以直接运行：

```powershell
$env:PYTHONPATH="src"
python -m asset_ledger.app
```

首次启动会在项目目录下创建 `data/设备资产台账.xlsx`。之后可以在设置页切换到其他工作簿。

## 日常使用

1. 点击“新增设备”，填写装备编码、设备名称和其他信息。
2. 在主列表使用搜索框或左侧条件查找设备。
3. 选择设备后，右侧可查看完整信息和变化记录。
4. 双击设备或点击“编辑”修改；填写修改说明可帮助以后追溯。
5. 在“设置”中维护分类、位置、状态、来源和当前操作者。

状态为停用的分类或位置不会再出现在新建设备选项中，但已有设备记录仍保留并可查看。

## Excel 工作簿

工作簿包含以下工作表：

- `资产台账`：每台设备一行，保存当前信息。
- `变更历史`：只追加字段级历史记录。
- `分类字典`：一级和二级设备分类。
- `位置字典`：位置层级。
- `枚举字典`：状态与来源选项。
- `系统配置`：工作簿版本、下一个资产序号和默认操作者。

每次写入前，程序会在工作簿旁的 `backups` 文件夹创建时间戳备份。

## 测试

标准库测试无需额外安装测试框架：

```powershell
$env:PYTHONPATH="src"
$env:QT_QPA_PLATFORM="offscreen"
python -m unittest discover -s tests -v
```

安装开发依赖后也可以运行：

```powershell
python -m pip install -r requirements-dev.txt
pytest
```
